Normas de presentación.

  • Un archivo .js por trabajo.
  • Encabezado obligatorio: Cada archivo debe incluir tu nombre, la fecha y la consigna a trabajar como comentario inicial.
  • Código documentado: Todo el código debe estar comentado. Explicá qué hace cada función y qué retorna.
  • Verificación previa: Probá el código en la consola del navegador antes de entregar.
  • Restricción de bucles: No se permiten bucles for, while ni variables globales mutables en las consignas.

Estructura mínima de cada archivo.

//===============================================================
// Consigna: [Nombre de la consigna]
// Alumno/a: [Tu nombre completo]
// Fecha: [dd/mm/aaaa]
// Descripción: [Qué hace este ejercicio en pocas palabras]
//===============================================================
